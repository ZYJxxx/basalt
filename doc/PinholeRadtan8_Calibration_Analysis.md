# Pinhole-Radtan8 相机模型单目标定流程分析

## 一、标定流程概述

### 1.1 初始化阶段

**代码位置**: `src/calibration/cam_calib.cpp::initCamIntrinsics()`

1. **内参初始化**（第533-646行）：
   - 使用 Zhang 方法通过单应性矩阵估计初始内参
   - 只初始化 `[fx, fy, cx, cy]` 四个参数
   - **所有畸变参数初始化为 0**（第700-707行）

```cpp
// thirdparty/basalt-headers/include/basalt/camera/pinhole_radtan8_camera.hpp:700
inline void setFromInit(const Vec4& init) {
  param_.setZero();  // 所有参数（包括畸变参数）初始化为0
  param_[0] = init[0];  // fx
  param_[1] = init[1];  // fy
  param_[2] = init[2];  // cx
  param_[3] = init[3];  // cy
  // k1-k6, p1-p2 全部为 0
  rpmax_ = S0;  // 无畸变，所以是双射的
}
```

2. **初始位姿估计**（第703-717行）：
   - 使用 RANSAC + PnP 估计每帧图像的相机位姿
   - 基于反投影误差进行内点筛选

### 1.2 优化阶段

**代码位置**: `include/basalt/optimization/poses_optimize.h::optimize()`

1. **Levenberg-Marquardt 优化**（第182-254行）：
   - 使用稀疏 Hessian 矩阵求解
   - 每次迭代更新所有参数（位姿、外参、内参）
   - 使用步长质量控制（step_quality）决定是否接受更新

2. **参数更新机制**（第203-206行）：
```cpp
for (size_t i = 0; i < calib->intrinsics.size(); i++) {
  auto &c = calib->intrinsics[i];
  c.applyInc(inc.segment(offset_cam_intrinsics[i], c.getN()));
}
```

3. **rpmax_ 动态更新**（第712-714行）：
```cpp
void operator+=(const VecN& inc) {
  param_ += inc;
  rpmax_ = computeRpmax();  // 每次参数更新后重新计算单射区域半径

```

## 二、畸变模型数学形式

### 2.1 畸变公式

**代码位置**: `thirdparty/basalt-headers/include/basalt/camera/pinhole_radtan8_camera.hpp:515-537`

```cpp
// 归一化坐标
const Scalar xp = x / z;
const Scalar yp = y / z;
const Scalar rp2 = xp * xp + yp * yp;

// 径向畸变系数（分数形式）
const Scalar cdist = (1 + rp2 * (k1 + rp2 * (k2 + rp2 * k3))) /
                     (1 + rp2 * (k4 + rp2 * (k5 + rp2 * k6)));

// 切向畸变
const Scalar deltaX = 2 * p1 * xp * yp + p2 * (rp2 + 2 * xp * xp);
const Scalar deltaY = 2 * p2 * xp * yp + p1 * (rp2 + 2 * yp * yp);

// 畸变后的坐标
const Scalar xpp = xp * cdist + deltaX;
const Scalar ypp = yp * cdist + deltaY;
```

### 2.2 参数数量

- **总参数数**: 12 个
  - 基础参数: `fx, fy, cx, cy` (4个)
  - 径向畸变: `k1, k2, k3, k4, k5, k6` (6个)
  - 切向畸变: `p1, p2` (2个)

## 三、畸变系数波动较大的原因分析

### 3.1 参数初始化问题

**问题**: 所有畸变参数从 0 开始优化

- **影响**: 没有先验信息，优化器需要从零开始探索参数空间
- **后果**: 容易陷入局部最优，不同运行可能得到不同的结果

**证据**（从 log 文件）:
```
初始化: Cam 0: 414.331 414.331 639.5 511.5 0 0 0 0 0 0 0 0
优化后: intrinsics 0: 501.105 501.178 620.132 513.14 0.3538 -0.172671 ...
```

### 3.2 参数强耦合

**问题**: 8 个畸变参数之间存在强耦合关系

1. **径向畸变参数耦合**:
   - `k1, k2, k3` 在分子中，`k4, k5, k6` 在分母中
   - 分子和分母的参数可以相互补偿，产生相似的重投影误差
   - 例如：`k1=0.1, k4=0.0` 和 `k1=0.2, k4=0.1` 可能产生相似的畸变效果

2. **数学上的等价性**:
   ```
   cdist = (1 + rp2*(k1 + rp2*(k2 + rp2*k3))) / (1 + rp2*(k4 + rp2*(k5 + rp2*k6)))
   ```
   - 如果同时缩放分子和分母的系数，可能得到相似的结果
   - 这导致 Hessian 矩阵接近奇异，优化不稳定

### 3.3 非线性畸变模型的复杂性

**问题**: 分数形式的畸变模型比多项式模型更复杂

1. **分母可能接近零**:
   - 当 `1 + rp2*(k4 + rp2*(k5 + rp2*k6))` 接近 0 时，畸变会急剧增大
   - 这导致优化过程中参数的小变化可能引起大的重投影误差变化

2. **梯度不稳定性**:
   - 雅可比矩阵的计算涉及复杂的链式求导
   - 参数空间的曲率变化剧烈，导致优化步长难以选择

### 3.4 rpmax_ 动态更新的影响

**问题**: 每次参数更新后都重新计算 `rpmax_`

**代码位置**: `pinhole_radtan8_camera.hpp:712-714`

```cpp
void operator+=(const VecN& inc) {
  param_ += inc;
  rpmax_ = computeRpmax();  // 重新计算单射区域半径
}
```

**影响**:
1. **计算开销**: `computeRpmax()` 使用梯度上升法，最多迭代 1000 次
2. **不连续性**: `rpmax_` 的变化可能导致某些点的投影状态突然改变（从有效变为无效，或反之）
3. **优化不稳定**: 这相当于在优化过程中动态改变约束条件

**证据**（从 log 文件）:
- 可以看到很多 `[REJECTED]` 步骤，说明优化过程不稳定
- `step_quality` 经常为负值，说明线性化模型预测不准确

### 3.5 优化过程中的不稳定性

**证据**（从 log 文件）:

**分析**:
- 多次 `[REJECTED]` 步骤说明优化器经常尝试不合理的参数更新
- `step_quality` 为负值说明实际误差增加，线性化模型预测失败
- 需要多次调整 `lambda` 才能找到可接受的步长

### 3.6 参数冗余和过拟合风险

**问题**: 8 个畸变参数对某些相机可能过多

1. **过拟合**: 过多的参数可能导致模型过度拟合噪声
2. **参数不确定性**: 参数之间的相关性导致单个参数的不确定性很大
3. **数值精度**: 参数的小变化可能被噪声掩盖，导致优化方向不确定

### 3.7 单射区域约束的影响

**代码位置**: `pinhole_radtan8_camera.hpp:348-349, 695`

```cpp
bool in_injective_area = rpmax_ == S0 ? true : rp2 <= rpmax_ * rpmax_;
bool is_valid = positive_z && in_injective_area;
```

**影响**:
- 当 `rpmax_` 变化时，某些点的投影状态会突然改变
- 这导致优化过程中有效点的数量动态变化
- 进一步增加了优化的不稳定性

## 五、改进建议

### 5.1 参数初始化改进

1. **使用先验信息**: 
   - 如果已知相机类型，可以使用该类型相机的典型畸变参数作为初始值
   - 或者使用 OpenCV 的标定结果作为初始值

2. **分阶段优化**:
   - 先优化基础参数（fx, fy, cx, cy）和低阶畸变参数（k1, k2, p1, p2）
   - 再优化高阶畸变参数（k3, k4, k5, k6）

### 5.2 参数约束

1. **添加正则化项**:
   - 对畸变参数添加 L2 正则化，防止参数过大
   - 或者使用参数边界约束

2. **参数归一化**:
   - 对畸变参数进行归一化，使不同参数的尺度相近

### 5.3 优化策略改进

1. **固定 rpmax_ 计算频率**:
   - 不要每次迭代都重新计算 `rpmax_`
   - 可以每隔几次迭代计算一次，或者只在参数变化较大时计算

2. **使用更稳定的优化器**:
   - 考虑使用 Trust Region 方法
   - 或者使用 Ceres Solver 的自动微分功能

### 5.4 模型简化

1. **自适应参数数量**:
   - 根据重投影误差决定是否需要所有 8 个畸变参数
   - 对于畸变较小的相机，可以使用更少的参数

2. **使用其他畸变模型**:
   - 对于畸变较小的相机，考虑使用更简单的模型（如 pinhole-radtan4）

## 六、总结

**pinhole-radtan8 模型畸变系数波动较大的主要原因**:

1. ✅ **参数初始化**: 所有畸变参数从 0 开始，没有先验信息
2. ✅ **参数强耦合**: 8 个畸变参数之间存在强耦合，导致 Hessian 矩阵接近奇异
3. ✅ **非线性复杂性**: 分数形式的畸变模型导致优化空间复杂
4. ✅ **rpmax_ 动态更新**: 每次参数更新后重新计算单射区域，增加不稳定性
5. ✅ **优化不稳定性**: 从 log 可以看到大量 REJECTED 步骤
6. ✅ **参数冗余**: 8 个参数可能对某些相机过多，导致过拟合和不确定性

这些因素共同作用，导致优化过程中参数波动较大，不同运行可能得到略有不同的标定结果。
